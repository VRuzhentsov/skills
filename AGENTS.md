# Skills Repository Instructions

## Pull Request Workflow

- Make every repository change on a feature branch and deliver it through a pull request.
- Do not commit or push changes directly to `main`.
- Before creating a branch, verify `main` is clean and current with `origin/main`.
- Branch from `origin/main`, keep commits scoped to the requested change, and inspect the final diff before opening a pull request.
- Do not merge a pull request without explicit user direction.
